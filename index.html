Tame It - Claim It

Seitenbau is present at the Google Devfest 2013 and we brought a little challenge with us. Master it and there's a good chance you'll soon be the owner of a fancy new smart watch!

The Task

You have to write an interpreter for the SFCPU - The Seitenbau Fantasy CPU. You can do that in whatever programming language you prefer. Just know that we have to be able to compile and run it (Meaning Turbo Pascal 6.0 targeting MS-DOS is not an option).

Send in your solution via email to coding-challenge@seitenbau.com. The DEADLINE is 05.11.13

SFCPU Specification

The information below should suffice to write an interpreter. If there is anything missing or you have further questions don't hesitate to contact us: coding-challenge@seitenbau.com

Memory And Registers

  • RAM : 256 bytes
  • A : 16bit general purpose register
  • B : 16bit general purpose register
  • C : 16bit general purpose register
  • Z : Zero Flag (Only modified by CMP Instruction)
  • IP : Instruction Pointer Register

All memory and registers are to be initialized to 0 on startup.

Instructions

Opcode Arguments Description
LOAD reg, value reg <- value ; IP+=1 ; Z=0;
LOADM reg, address reg <- ram[address] ; IP+=1 ; Z=0;
SETM address, reg ram[address] <- reg ; IP+=1 ; Z=0;
ADD reg1, *reg2 reg3 reg1 <- reg2 + reg3 ; Z=0;
SUB reg1, reg2 reg3 reg1 <- reg2 - reg3 ; Z=0;
INC reg reg <- reg+1 ; IP+=1 ; Z=0;
DEC reg reg <- reg-1 ; IP+=1 ; Z=0;
OR reg, value reg1 <- reg2
XOR reg, value reg <- reg ^ value ; IP+=1 ; Z=0;
AND reg, value reg <- reg & value ; IP+=1 ; Z=0;
CMP reg, value Z <- (reg == value) ? 1 : 0 ; IP+=1;
JNE offset IP <- (Z == 0) ? IP + offset : IP + 1 ; Z=0;
PRINT register prints reg value IP+=1 ; Z=0; (no linebreak)
STOP stop execution ; Z=0;

NOTES

If anything odd happens at runtime always trigger a STOP instruction. Running out of code also qualifies as odd - this means your programs will implicitly always end with a STOP instruction . The STOP instruction should print the following (including the leading line break):


A: <register-A>,
B: <register-B>,
C: <register-C>,
Z: <register-Z>
IP: <register-IP>

Example Code

LOAD A, 1
SETM 0, A
LOADM A,0
OR A, 8
PRINT A
XOR A, 42
PRINT A
XOR A, 42
PRINT A
LOAD C, 10
PRINT C
DEC C
CMP C, 0
JNE -3
STOP

The code above should print the following:

935910987654321

A: 9
B: 0
C: 0
Z: true
IP: 15

Rules & General

  1. Use any programming language
  2. Your program should take exactly one argument (the input file) and print any output to stdout.
  3. Your program should exit with code 1 when parsing the input failed. Otherwise exit with 0
  4. We may throw wacky input at your program, so be prepared!
  5. Above all: have fun and be creative!
